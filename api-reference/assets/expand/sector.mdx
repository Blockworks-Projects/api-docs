## Overview

Add the `?expand=sector` flag to your `/assets` or `/assets/{idOrSlug}` page to include **sector** data in the response.

## Example Request

<CodeGroup>

```bash cURL
curl -H "x-api-key: YOUR_API_KEY" \
  "https://api.blockworks.com/v1/assets/ethereum?expand=sector"
```

```typescript TypeScript
const res = await fetch(
  'https://api.blockworks.com/v1/assets/ethereum?expand=sector',
  { headers: { 'x-api-key': 'YOUR_API_KEY' } }
)
const data = await res.json()
console.log(data.sector)
```

```python Python
import requests

r = requests.get(
  'https://api.blockworks.com/v1/assets/ethereum',
  headers={'x-api-key': 'YOUR_API_KEY'},
  params={'expand': 'sector'}
)
data = r.json()
print(data.get('sector'))
```

</CodeGroup>

### Field Reference

| Field | Type | Description |
|---|---|---|
| `id` | number | Unique sector identifier |
| `name` | string | Sector name (e.g., Smart Contract Platform) |

## Notes

- Can be used with other expansion options by joining with a comma (e.g. `?expand=markets,ohlcv_last_24_h`).
- Can be used on `/v1/assets/` or `/v1/assets/{idOrSlug}` endpoints.
- `price` and `market_cap` are included by default in asset results.

## Overview

Add the `?expand=supply` flag to your `/assets` or `/assets/{idOrSlug}` page to include **supply** data in the response.

## Example Request

<CodeGroup>

```bash cURL
curl -H "x-api-key: YOUR_API_KEY" \
  "https://api.blockworks.com/v1/assets/ethereum?expand=supply"
```

```typescript TypeScript
const res = await fetch(
  'https://api.blockworks.com/v1/assets/ethereum?expand=supply',
  { headers: { 'x-api-key': 'YOUR_API_KEY' } }
)
const data = await res.json()
console.log(data.supply)
```

```python Python
import requests

r = requests.get(
  'https://api.blockworks.com/v1/assets/ethereum',
  headers={'x-api-key': 'YOUR_API_KEY'},
  params={'expand': 'supply'}
)
data = r.json()
print(data.get('supply'))
```

</CodeGroup>

### Field Reference

| Field | Type | Description |
|---|---|---|
| `asset_id` | number | Unique identifier for the asset |
| `circulating` | number | Number of tokens currently in circulation |
| `liquid` | number | Liquid supply amount |
| `total` | number | Total token supply including locked tokens |

## Notes

- Can be used with other expansion options by joining with a comma (e.g. `?expand=markets,ohlcv_last_24_h`).
- Can be used on `/v1/assets/` or `/v1/assets/{idOrSlug}` endpoints.
- `price` and `market_cap` are expanded by default in `/assets/{idOrSlug}` results unless an explicit `?expand=` flag is provided.

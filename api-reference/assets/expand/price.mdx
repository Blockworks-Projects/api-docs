## Overview

Add the `?expand=price` flag to your `/assets` or `/assets/{idOrSlug}` page to include **price** data in the response.

## Example Request

<CodeGroup>

```bash cURL
curl -H "x-api-key: YOUR_API_KEY" \
  "https://api.blockworks.com/v1/assets/ethereum?expand=price"
```

```typescript TypeScript
const res = await fetch(
  'https://api.blockworks.com/v1/assets/ethereum?expand=price',
  { headers: { 'x-api-key': 'YOUR_API_KEY' } }
)
const data = await res.json()
```

```python Python
import requests

r = requests.get(
  'https://api.blockworks.com/v1/assets/ethereum',
  headers={'x-api-key': 'YOUR_API_KEY'},
  params={'expand': 'price'}
)
data = r.json()
```

</CodeGroup>

## Example Response

```json
{
  "id": 2,
  "code": "ETH",
  "title": "Ethereum",
  "slug": "ethereum",
  ... // other asset data
  "price": {
    "usd": 4473.71,
    "btc": 0.051222333099357,
    "eth": 1,
    "updated_at": 1759454319
  }
}
```

### Response Fields

| Field | Type | Description |
|---|---|---|
| `usd` | number | Current price in US dollars |
| `btc` | number | Current price in Bitcoin |
| `eth` | number | Current price in Ethereum |
| `updated_at` | number | Unix timestamp when the price was last updated |

## Notes

- Can be used with other expansion options by joining with a comma (e.g. `?expand=markets,ohlcv_last_24h`).
- Can be used on `/v1/assets/` or `/v1/assets/{idOrSlug}` endpoints.
- `price` and `market_cap` are expanded by default in `/assets/{idOrSlug}` results unless an explicit `?expand=` flag is provided.

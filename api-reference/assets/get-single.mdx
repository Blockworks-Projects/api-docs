## Overview

Fetch a single asset by **ID** or **slug**. You can optionally expand related objects (e.g. supply) to avoid making multiple requests. Single asset payloads include `price` and `market_cap` expansions by default, unless an explicit `?expand=` flag is provided.

## Basic Example

<CodeGroup>

```bash cURL
curl -H "x-api-key: YOUR_API_KEY" \\
  "https://api.blockworks.com/v1/assets/eth?expand=supply"
```

```typescript TypeScript
const res = await fetch(
  'https://api.blockworks.com/v1/assets/eth?expand=supply',
  { headers: { 'x-api-key': 'YOUR_API_KEY' } }
)
const data = await res.json()
```

```python Python
import requests

r = requests.get(
  'https://api.blockworks.com/v1/assets/eth',
  headers={'x-api-key': 'YOUR_API_KEY'},
  params={'expand': 'supply'}
)
data = r.json()
```

</CodeGroup>

## Supported Options

| Name | Type | Details |
| - | - | - |
| `idOrSlug` | path | The ID or slug of the asset to fetch |
| `expand` | query | The fields to expand. See **Expand Options** for more details. |

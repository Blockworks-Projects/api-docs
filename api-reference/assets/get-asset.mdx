## GET /v3/assets/{idOrSlug}

Retrieve detailed information about a specific crypto asset by its ID or slug.

### Response Structure

The API returns a single asset object with the following structure:

```json
{
  "id": 2,
  "code": "ETH",
  "title": "Ethereum",
  "slug": "ethereum",
  "description": "Ethereum is a distributed blockchain...",
  "sector_id": 40,
  // ... additional core fields
  // Expanded data appears here when requested
}
```

### Request

```bash
GET https://rest.staging.blockworksresearch.com/v3/assets/{idOrSlug}
```

### Path Parameters

| Parameter | Type | Description | Required |
|-----------|------|-------------|----------|
| `idOrSlug` | string | Asset ID or slug identifier | Yes |

### Authentication

<CodeGroup>

```bash cURL
curl -H "x-api-key: YOUR_API_KEY" \
  "https://rest.staging.blockworksresearch.com/v3/assets/bitcoin"
```

```typescript TypeScript
const data = await fetch('https://rest.staging.blockworksresearch.com/v3/assets/bitcoin', {
  headers: {
    'x-api-key': 'YOUR_API_KEY'
  }
})
.then(r => r.json());
```

```python Python

headers = {
    'x-api-key': 'YOUR_API_KEY',
}

response = requests.get(
    'https://rest.staging.blockworksresearch.com/v3/assets/bitcoin',
    headers=headers
)
```

</CodeGroup>

### Query Parameters

| Parameter | Type | Description | Default |
|-----------|------|-------------|---------|
| `expand` | array | Expand related data | - |

### Expansion Options

Use the `expand` parameter to include additional related data. Multiple fields can be expanded by separating them with commas:

| Value | Description | Data Type |
|-------|-------------|-----------|
| `price` | Current price in USD, BTC, ETH with 7-day sparkline | Object |
| `market_cap` | Market cap, rank, dominance, and percentage changes | Object |
| `sector` | Sector classification with ID and title | Object |
| `addresses` | Contract addresses across different chains | Array |
| `markets` | Trading market data including volume and ATH | Array |
| `supply` | Circulating, liquid, and total supply information | Object |
| `ohlcv_last_24_h` | OHLCV (Open, High, Low, Close, Volume) data | Object |
| `reference` | External references (CoinGecko, Messari, etc.) | Object |
| `addresses.chain` | Address data with expanded chain information | Array |
| `is_favorite` | Whether the asset is marked as favorite | Boolean |

### Response Fields

| Field | Type | Description |
|-------|------|-------------|
| `id` | number | Unique asset identifier |
| `code` | string | Asset code/symbol (e.g., "BTC", "ETH") |
| `title` | string | Full asset name |
| `slug` | string | URL-friendly asset identifier |
| `tag_line` | string | Brief description of the asset |
| `description` | string | Detailed asset description |
| `image_url` | string | URL to asset logo/image |
| `legacy_sector` | string | Legacy sector classification |
| `category` | string | Asset category (Infrastructure, Financial, etc.) |
| `is_supported` | boolean | Whether asset is supported |
| `updated_at` | number | Unix timestamp of last update |
| `sector_id` | number | Sector identifier |
| `type` | string | Asset type (Infrastructure, Application, etc.) |
| `sector` | object | Sector information (if expanded) |
| `market_cap` | object | Market cap data with USD value, rank, changes (if expanded) |
| `price` | object | Price data in USD, BTC, ETH with sparkline (if expanded) |
| `supply` | object | Supply information: circulating, liquid, total (if expanded) |
| `ohlcv_last_24_h` | object | 24-hour OHLCV data (if expanded) |
| `reference` | object | External references to CoinGecko, Messari, etc. (if expanded) |
| `addresses` | array | Contract addresses across chains (if expanded) |
| `markets` | array | Trading market data including volume and ATH (if expanded) |
| `is_favorite` | boolean | Whether marked as favorite (if expanded) |

### Common Use Cases

#### Asset Overview Dashboard

Get comprehensive asset information for a dashboard:

```typescript
async function getAssetOverview(slug: string) {
  const data = await fetch(`https://rest.staging.blockworksresearch.com/v3/assets/${slug}?expand=price,market_cap,sector,metrics`, {
    headers: { 'x-api-key': 'YOUR_API_KEY' }
  });
  
  const asset = data.data;
  
  return {
    name: asset.name,
    symbol: asset.symbol,
    price: asset.price,
    marketCap: asset.market_cap,
    sector: asset.sector,
    priceChange24h: asset.metrics.price_change_24h,
    volume24h: asset.metrics.volume_24h,
    rank: asset.metrics.market_cap_rank
  };
}

// Usage
const bitcoinOverview = await getAssetOverview('bitcoin');
```

#### Portfolio Asset Details

Get detailed information for portfolio assets:

```python
def get_portfolio_asset_details(assets: list):
    """Get detailed information for portfolio assets"""
    portfolio_details = []
    
    for asset_slug in assets:
        response = requests.get(
            f'https://rest.staging.blockworksresearch.com/v3/assets/{asset_slug}',
            headers={'x-api-key': 'YOUR_API_KEY'},
            params={'expand': 'price,market_cap,metrics,addresses'}
        )
        
        if response.status_code == 200:
            asset_data = response.json()['data']
            portfolio_details.append({
                'name': asset_data['name'],
                'symbol': asset_data['symbol'],
                'price': asset_data['price'],
                'market_cap': asset_data['market_cap'],
                'volatility': asset_data['metrics']['volatility_30d'],
                'addresses': asset_data['addresses']
            })
    
    return portfolio_details

# Usage
portfolio_assets = ['bitcoin', 'ethereum', 'solana']
details = get_portfolio_asset_details(portfolio_assets)
```

#### Market Data Analysis

Analyze trading markets for an asset:

```python
def analyze_asset_markets(slug: str):
    """Analyze trading markets for an asset"""
    response = requests.get(
        f'https://rest.staging.blockworksresearch.com/v3/assets/{slug}',
        headers={'x-api-key': 'YOUR_API_KEY'},
        params={'expand': 'markets,price,metrics'}
    )
    
    if response.status_code == 200:
        asset_data = response.json()['data']
        
        # Sort markets by volume
        markets = sorted(asset_data['markets'], 
                        key=lambda x: x['volume_24h'], 
                        reverse=True)
        
        print(f"Top markets for {asset_data['name']}:")
        for market in markets[:5]:
            print(f"  {market['exchange']}: {market['pair']} - ${market['volume_24h']:,.0f}")
        
        return markets
    
    return None

# Usage
btc_markets = analyze_asset_markets('bitcoin')
```

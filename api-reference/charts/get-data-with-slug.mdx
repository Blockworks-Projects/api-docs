## Overview

Fetch chart data using both **dashboard slug** and **visualization slug**. This endpoint provides a more descriptive way to access chart data when you know both the dashboard and specific visualization identifiers.

## Example Request

<CodeGroup>

```bash cURL
curl -H "x-api-key: YOUR_API_KEY" \
  "https://api.blockworks.com/v1/charts/5/staking-data/data?limit=5"
```

```typescript TypeScript
const res = await fetch(
  'https://api.blockworks.com/v1/charts/5/staking-data/data?limit=5',
  { headers: { 'x-api-key': 'YOUR_API_KEY' } }
)
const data = await res.json()
```

```python Python
import requests

r = requests.get(
  'https://api.blockworks.com/v1/charts/5/staking-data/data',
  headers={'x-api-key': 'YOUR_API_KEY'},
  params={'limit': 5, 'order_by': 'dt', 'order_dir': 'desc'}
)
data = r.json()
```

</CodeGroup>

## Supported Options

| Name | Type | Details |
| - | - | - |
| `visualizationIdOrDashboardSlug` | path | The dashboard slug or visualization ID. |
| `visualizationSlug` | path | The specific visualization slug within the dashboard. |
| `limit` | query | Number of data points to return (1-100000, default: 10000). |
| `page` | query | Page number for pagination (default: 1). |
| `search` | query | Search term to filter data. |
| `select` | query | Array of column names to include in response. |
| `order_by` | query | Column name to sort by. |
| `order_dir` | query | Sort direction: "asc" or "desc" (default: "asc"). |

## Notes

- This endpoint provides the same functionality as the single-parameter version but with more semantic URL structure.
- Useful when working with dashboard-organized visualizations.
- Both slug components are required in the URL path.

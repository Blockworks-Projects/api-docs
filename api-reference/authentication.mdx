## Authentication Overview

The Blockworks Research API uses API keys for authentication. All API requests must include a valid API key to access the endpoints.

## API Key Authentication

### Using the x-api-key Header (Recommended)

Include your API key in the `x-api-key` header:

<CodeGroup>

```bash cURL
curl -H "x-api-key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  https://rest.staging.blockworksresearch.com/v3/assets
```

```typescript TypeScript
const response = await fetch('https://rest.staging.blockworksresearch.com/v3/assets', {
  headers: {
    'x-api-key': 'YOUR_API_KEY',
    'Content-Type': 'application/json'
  }
});
```

```python Python
import requests

headers = {
    'x-api-key': 'YOUR_API_KEY',
    'Content-Type': 'application/json'
}

response = requests.get(
    'https://rest.staging.blockworksresearch.com/v3/assets',
    headers=headers
)
```

</CodeGroup>

### Using Bearer Token Authentication

Alternatively, you can use the `Authorization` header with Bearer token:

<CodeGroup>

```bash cURL
curl -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  https://rest.staging.blockworksresearch.com/v3/assets
```

```typescript TypeScript
const response = await fetch('https://rest.staging.blockworksresearch.com/v3/assets', {
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Content-Type': 'application/json'
  }
});
```

```python Python
import requests

headers = {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Content-Type': 'application/json'
}

response = requests.get(
    'https://rest.staging.blockworksresearch.com/v3/assets',
    headers=headers
)
```

</CodeGroup>

## Authentication Errors

### 401 Unauthorized

This error occurs when your API key is invalid or missing:

```json
{
  "error": {
    "code": "UNAUTHORIZED",
    "message": "Invalid API key"
  }
}
```

**Common causes:**
- Missing API key in the request
- Invalid or expired API key
- Incorrect header format

### 403 Forbidden

This error occurs when your API key doesn't have permission to access a resource:

```json
{
  "error": {
    "code": "FORBIDDEN",
    "message": "Insufficient permissions for this resource"
  }
}
```

**Common causes:**
- Your plan doesn't include access to this endpoint
- API key has been restricted
- Resource requires higher permission level

### Environment Variables Example

<CodeGroup>

```bash .env
# Store in .env file
BLOCKWORKS_API_KEY=your_api_key_here
BLOCKWORKS_API_URL=https://rest.staging.blockworksresearch.com
```

```typescript TypeScript
// Load from environment
const apiKey = process.env.BLOCKWORKS_API_KEY;
const apiUrl = process.env.BLOCKWORKS_API_URL;

if (!apiKey) {
  throw new Error('BLOCKWORKS_API_KEY is required');
}

const response = await fetch(`${apiUrl}/v3/assets`, {
  headers: {
    'x-api-key': apiKey,
    'Content-Type': 'application/json'
  }
});
```

```python Python
import os
from dotenv import load_dotenv

load_dotenv()

api_key = os.getenv('BLOCKWORKS_API_KEY')
api_url = os.getenv('BLOCKWORKS_API_URL')

if not api_key:
    raise ValueError('BLOCKWORKS_API_KEY is required')

headers = {
    'x-api-key': api_key,
    'Content-Type': 'application/json'
}
```

</CodeGroup>

## Rate Limiting

API keys are subject to rate limits based on your subscription plan:

| Plan | Requests per Hour | Requests per Day |
|------|-------------------|------------------|
| Starter | 1,000 | 10,000 |
| Professional | 10,000 | 100,000 |
| Enterprise | Custom | Custom |

Rate limit information is included in response headers:

```
X-RateLimit-Limit: 1000
X-RateLimit-Remaining: 999
X-RateLimit-Reset: 1640995200
```

### Rate Limit Exceeded

When you exceed your rate limit, you'll receive a 429 error:

```json
{
  "error": {
    "code": "RATE_LIMIT_EXCEEDED",
    "message": "Rate limit exceeded",
    "details": {
      "limit": 1000,
      "remaining": 0,
      "reset": 1640995200
    }
  }
}
```

## Testing Your Authentication

Test your API key with a simple request:

<CodeGroup>

```bash cURL
curl -H "x-api-key: YOUR_API_KEY" \
  https://rest.staging.blockworksresearch.com/v3/assets?limit=1
```

```typescript TypeScript
async function testAuth() {
  try {
    const response = await fetch('https://rest.staging.blockworksresearch.com/v3/assets?limit=1', {
      headers: {
        'x-api-key': 'YOUR_API_KEY',
        'Content-Type': 'application/json'
      }
    });
    
    if (response.ok) {
      console.log('✅ Authentication successful');
      const data = await response.json();
      console.log('Sample data:', data);
    } else {
      console.error('❌ Authentication failed:', response.status);
    }
  } catch (error) {
    console.error('❌ Request failed:', error);
  }
}

testAuth();
```

```python Python
import requests

def test_auth():
    try:
        headers = {
            'x-api-key': 'YOUR_API_KEY',
            'Content-Type': 'application/json'
        }
        
        response = requests.get(
            'https://rest.staging.blockworksresearch.com/v3/assets?limit=1',
            headers=headers
        )
        
        if response.status_code == 200:
            print('✅ Authentication successful')
            data = response.json()
            print('Sample data:', data)
        else:
            print(f'❌ Authentication failed: {response.status_code}')
            print(response.text)
    except Exception as error:
        print(f'❌ Request failed: {error}')

test_auth()
```

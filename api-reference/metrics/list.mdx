## GET /v3/metrics

Retrieve a paginated list of available metrics that can be queried for crypto assets and protocols.

### Response Structure

The API returns a paginated response with the following structure:

```json
{
  "data": [
    {
      "name": "Active Addresses",
      "description": "Not a measure of user growth",
      "identifier": "active-addresses",
      "project": "Avax",
      "source": "Blockworks",
      "data_type": "timeseries_int",
      "parameters": { /* available query parameters */ }
    }
  ],
  "total": 12,
  "page": 1
}
```

### Request

```bash
GET https://rest.staging.blockworksresearch.com/v3/metrics
```

### Authentication

<CodeGroup>

```bash cURL
curl -H "x-api-key: YOUR_API_KEY" \
  "https://rest.staging.blockworksresearch.com/v3/metrics"
```

```typescript TypeScript
const data = await fetch('https://rest.staging.blockworksresearch.com/v3/metrics', {
  headers: {
    'x-api-key': 'YOUR_API_KEY',
  }
});
```

```python Python
import requests

headers = {
    'x-api-key': 'YOUR_API_KEY',
}

response = requests.get(
    'https://rest.staging.blockworksresearch.com/v3/metrics',
    headers=headers
)
```

</CodeGroup>

### Query Parameters

| Parameter | Type | Description | Default |
|-----------|------|-------------|---------|
| `page` | integer | Page number (1-indexed) | 1 |
| `limit` | integer | Items per page (max 100) | 10 |
| `project` | string | Filter by project identifier | - |
| `category` | string | Filter by metric category | - |
| `search` | string | Search metrics by name or description | - |

### Examples

#### Basic Request

<CodeGroup>

```bash cURL
curl -H "x-api-key: YOUR_API_KEY" \
  "https://rest.staging.blockworksresearch.com/v3/metrics?limit=20"
```

```typescript TypeScript
const data = await fetch('https://rest.staging.blockworksresearch.com/v3/metrics?limit=20', {
  headers: {
    'x-api-key': 'YOUR_API_KEY',
  }
})
.then(r => r.json());
console.log(`Found ${data.data.length} metrics`);

data.data.forEach(metric => {
  console.log(`${metric.name}: ${metric.description}`);
});
```

```python Python
import requests

headers = {'x-api-key': 'YOUR_API_KEY'}
params = {'limit': 20}

response = requests.get(
    'https://rest.staging.blockworksresearch.com/v3/metrics',
    headers=headers,
    params=params
)

data = response.json()
print(f"Found {len(data['data'])} metrics")

for metric in data['data']:
    print(f"{metric['name']}: {metric['description']}")
```

</CodeGroup>

#### Search Metrics

<CodeGroup>

```bash cURL
curl -H "x-api-key: YOUR_API_KEY" \
  "https://rest.staging.blockworksresearch.com/v3/metrics?search=volume"
```

```typescript TypeScript
const data = await fetch('https://rest.staging.blockworksresearch.com/v3/metrics?search=volume', {
  headers: {
    'x-api-key': 'YOUR_API_KEY',
  }

.then(r => r.json());
console.log(`Found ${data.data.length} volume-related metrics`);

data.data.forEach(metric => {
  console.log(`${metric.identifier}: ${metric.name}`);
  console.log(`  Unit: ${metric.unit}`);
  console.log(`  Projects: ${metric.projects.join(', ')}`);
});
```

```python Python
headers = {'x-api-key': 'YOUR_API_KEY'}
params = {'search': 'volume'}

response = requests.get(
    'https://rest.staging.blockworksresearch.com/v3/metrics',
    headers=headers,
    params=params
)

data = response.json()
print(f"Found {len(data['data'])} volume-related metrics")

for metric in data['data']:
    print(f"{metric['identifier']}: {metric['name']}")
    print(f"  Unit: {metric['unit']}")
    print(f"  Projects: {', '.join(metric['projects'])}")
```

</CodeGroup>

### Response Fields

| Field | Type | Description |
|-------|------|-------------|
| `name` | string | Human-readable metric name |
| `description` | string | Detailed metric description |
| `identifier` | string | Unique metric identifier |
| `project` | string | Project this metric belongs to |
| `source` | string | Data source (e.g., "Blockworks") |
| `data_type` | string | Data type (e.g., "timeseries_int", "timeseries_usd") |
| `parameters` | object | Available query parameters |
| `updated_at` | number | Unix timestamp of last update |
| `total` | number | Total number of metrics available |
| `page` | number | Current page number |

### Metric Categories

The API organizes metrics into several categories:

| Category | Description | Examples |
|----------|-------------|----------|
| `defi` | DeFi protocol metrics | TVL, fees, liquidity |
| `trading` | Trading-related metrics | Volume, spreads, trades |
| `network` | Network activity metrics | Users, transactions, gas |
| `token` | Token-specific metrics | Supply, distribution, holders |
| `governance` | Governance metrics | Proposals, voting, participation |

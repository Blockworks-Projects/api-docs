## Authentication

All API endpoints require authentication. We support multiple authentication methods:

### API Key (Recommended)

Include your API key in the `x-api-key` header:

```bash
curl -H "x-api-key: YOUR_API_KEY" \
  https://rest.staging.blockworksresearch.com/v3/assets
```

### Bearer Token

Alternatively, use Bearer token authentication:

```bash
curl -H "Authorization: Bearer YOUR_TOKEN" \
  https://rest.staging.blockworksresearch.com/v3/assets
```

## Response Format

All API responses follow a consistent structure:

```json
{
  "data": [...],
  "pagination": {
    "page": 1,
    "limit": 10,
    "total": 1000,
    "pages": 100
  }
}
```

### Data Field

The `data` field contains the actual response data:
- For list endpoints: An array of objects
- For single resource endpoints: A single object

### Pagination

List endpoints include pagination information:
- `page`: Current page number (1-indexed)
- `limit`: Number of items per page
- `total`: Total number of items
- `pages`: Total number of pages

## Query Parameters

### Pagination Parameters

| Parameter | Type | Description | Default |
|-----------|------|-------------|---------|
| `page` | integer | Page number (1-indexed) | 1 |
| `limit` | integer | Items per page (max 100) | 10 |

### Expansion Parameters

Use the `expand` parameter to include additional related data:

```bash
curl -H "x-api-key: YOUR_API_KEY" \
  "https://rest.staging.blockworksresearch.com/v3/assets?expand=price,market_cap,sector"
```

## Error Handling

The API uses standard HTTP status codes:

| Status Code | Description |
|-------------|-------------|
| 200 | Success |
| 400 | Bad Request - Invalid parameters |
| 401 | Unauthorized - Invalid API key |
| 404 | Not Found - Resource doesn't exist |
| 429 | Too Many Requests - Rate limit exceeded |
| 500 | Internal Server Error |

Error responses include details about the issue:

```json
{
  "error": {
    "code": "INVALID_PARAMETER",
    "message": "The 'expand' parameter contains invalid values",
    "details": {
      "invalid_values": ["invalid_field"]
    }
  }
}
```

## Rate Limits

API requests are subject to rate limits based on your plan:

| Plan | Requests per Hour | Requests per Day |
|------|-------------------|------------------|
| Starter | 1,000 | 10,000 |
| Professional | 10,000 | 100,000 |
| Enterprise | Custom | Custom |

Rate limit headers are included in all responses:

```
X-RateLimit-Limit: 1000
X-RateLimit-Remaining: 999
X-RateLimit-Reset: 1640995200
```

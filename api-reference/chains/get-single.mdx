## Overview

Fetch a single chain by **ID** (1), or **slug** (`ethereum`). You can optionally include sparkline data to visualize recent trends. This endpoint provides comprehensive information about a specific blockchain network.

## Example Request

<CodeGroup>

```bash cURL
curl -H "x-api-key: YOUR_API_KEY" \
  "https://api.blockworks.com/v1/chains/ethereum"
```

```typescript TypeScript
const res = await fetch(
  'https://api.blockworks.com/v1/chains/ethereum',
  { headers: { 'x-api-key': 'YOUR_API_KEY' } }
)
const data = await res.json()
```

```python Python
import requests

r = requests.get(
  'https://api.blockworks.com/v1/chains/ethereum',
  headers={'x-api-key': 'YOUR_API_KEY'}
)
data = r.json()
```

</CodeGroup>

## Supported Options

| Name | Type | Details |
| - | - | - |
| `idOrSlug` | path | The ID or slug of the chain to fetch |
| `sparkline` | query | Include sparkline data (e.g., '7d' for 7-day price history) |

## Response Fields

| Field | Type | Description |
|---|---|---|
| `id` | number | Unique chain identifier |
| `project_id` | number \| null | Associated project ID |
| `sector_id` | number \| null | Associated sector ID |
| `network_id` | number \| null | Associated network ID |
| `chain_identifier` | string \| null | Chain identifier |
| `title` | string | Full chain name |
| `slug` | string | URL-friendly identifier |
| `type` | string | Chain type classification |
| `image_url` | string \| null | URL to chain logo/image |
| `rpcs` | string \| null | RPC endpoints |
| `is_test_net` | number | Whether this is a testnet (0 = mainnet, 1 = testnet) |
| `updated_at` | number | Unix timestamp of last update |
| `asset` | object \| null | Associated asset with full details including market_cap, price, markets, and supply data |
